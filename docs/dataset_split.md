# 训练集与验证集划分说明

> 最优化理论 Project 3 (2026) · 数据预处理归档文档

## 1. 概述

本文档记录 iris 与 MNIST 两个数据集的训练集 / 验证集划分方法与结果。划分由脚本
`src/data_split.py` 自动完成，**固定随机种子**，保证全队成员、多次运行得到完全
一致且可复现的划分结果。

## 2. 数据集说明

| 数据集 | 规模 | 特征 / 维度 | 类别 | 原始文件 |
|--------|------|-------------|------|----------|
| iris | 150 样本 | 4 个数值特征 | 3 类，每类 50 个 | `iris/iris.data` |
| MNIST | 训练 60000 + 测试 10000 | 28×28 灰度图 | 10 类（数字 0–9） | `mnist/train-*`、`mnist/t10k-*` |

## 3. 划分方法

- **划分方式**：分层抽样（stratified sampling）。每个类别独立按比例抽取，保证
  训练集 / 验证集中各类别比例与原始数据严格一致。
- **划分比例**：训练集 80% / 验证集 20%。
- **随机种子**：`SEED = 42`（`numpy.random.default_rng(42)`），保证可复现。
- **MNIST 范围**：仅对原始的 **60000 个训练样本**进行划分；10000 个 t10k 样本
  作为**最终测试集**保留，不参与本次划分。
- 划分后的训练 / 验证索引整体打乱，可直接按顺序使用。

## 5. 产出文件

全部产出位于 `splits/` 目录，已纳入 Git，全队共用：

| 文件 | 内容 |
|------|------|
| `splits/iris_train.csv` | iris 训练集 120 条（含表头） |
| `splits/iris_val.csv` | iris 验证集 30 条（含表头） |
| `splits/mnist_train_indices.npy` | MNIST 训练集索引（48000 个 int32，指向原始 60000 训练集） |
| `splits/mnist_val_indices.npy` | MNIST 验证集索引（12000 个 int32） |
| `splits/split_summary.json` | 划分参数与各类别统计 |

> **为什么 MNIST 用索引而非图像副本？** 直接保存图像副本会重复占用约 40 MB 空间。
> 保存索引（指向原始训练集的行号）仅需数百 KB，且加载时配合原始 `mnist/train-*`
> 文件即可还原，更利于版本控制。

## 6. 复现与使用

### 复现划分

```bash
uv run python src/data_split.py
```

如需修改比例或种子，编辑 `src/data_split.py` 顶部的 `SEED` / `VAL_RATIO` 后重跑，
并同步更新本文档。

### 加载划分（Python）

```python
import sys
sys.path.append("src")
from data_split import load_iris_split, load_mnist_split

# iris：返回 (特征矩阵 X, 标签 y)
X_train, y_train = load_iris_split("train")   # X_train 形状 (120, 4)
X_val,   y_val   = load_iris_split("val")     # X_val   形状 (30, 4)

# MNIST：返回 (图像, 标签)，图像形状 (N, 28, 28)
img_train, lab_train = load_mnist_split("train")   # 48000 张
img_val,   lab_val   = load_mnist_split("val")     # 12000 张
```

## 7. 注意事项

- 划分文件已纳入 Git，**全队共用同一份划分**，请勿各自重新划分，否则实验结果不可比。
- t10k 测试集仅用于**最终评估**，训练与调参阶段不得使用，以免信息泄漏。
- MNIST 索引指向原始训练集的行号，加载时必须配合原始 `mnist/train-*` 文件。
- 修改划分逻辑后，请重新运行脚本并更新本文档第 4 节的统计结果。
